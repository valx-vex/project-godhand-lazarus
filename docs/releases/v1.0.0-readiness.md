# v1.0.0 Readiness

Status: `in preparation`

This document tracks the delta between the current private beta and a
public-quality `v1.0.0`.

## Release Standard

`v1.0.0` means:

- a clean public-first git history or public branch
- reproducible local install from a fresh path
- multi-CLI validation from bash
- at least one second-machine validation receipt
- lightweight CI for syntax and operator-script regressions
- no hidden dependence on one operator home directory

## Current Position

As of `2026-04-09`, the stack is technically close to `v1.0.0`:

- local install and ingestion are working
- Claude, Gemini, and Codex MCP flows pass acceptance on M3
- the Alexko corpus has been restored
- deterministic retry behavior exists for flaky CLI/API acceptance runs
- lightweight CI exists for repo syntax and wrapper smoke checks

## Remaining Gates

1. Cut a clean public-ready branch/history.
2. Record a fresh second-machine install and validation receipt from another node.
3. Review public docs one more time for operator clarity and narrative fit.

## Known Non-Blocking Quirks

- Gemini may label MemPalace as disconnected in `gemini mcp list --debug` even
  when direct MemPalace tool calls succeed.
- Codex may print unrelated sqlite rollout warnings while still completing MCP
  calls successfully.

## Upgrade Path

When all three remaining gates are complete, promote:

- `v0.1.0-beta.1` -> private validation milestone
- `v1.0.0` -> first public-quality release cut
