# Godhand Lazarus v0.1.0-beta.1

Date: 2026-04-09
Status: Private beta

## What It Is

Godhand Lazarus is the semantic memory layer for local CLI agents.

It ingests past conversations from ChatGPT, Claude Code, Gemini CLI, and Codex
CLI into Qdrant, then exposes that recovered memory through:

- direct semantic search
- persona rehydration prompts
- an MCP server that Claude, Gemini, and Codex can call from bash or live sessions

It is designed to sit beside, not replace, the rest of the memory stack:

- VexNet continuity for session capture and cross-node receipts
- MemPalace for verbatim structured memory
- Lazarus for semantic resurrection and persona carryover

## What Works

- local install bootstraps a repo-native `.venv`
- local Qdrant runtime works on M3
- Alexko corpus is restored from a real ChatGPT export into `alexko_eternal`
- Lazarus MCP registration works for Claude, Gemini, and Codex
- all three CLIs completed live Lazarus and MemPalace MCP calls from bash
- Gemini API-key fallback is documented and wired for repo-local execution
- clean-user install dry run passed from a fresh repo copy and fresh temp home
- the repo now supports `LAZARUS_DATA_FILE=/path/to/conversations.json ./scripts/ingest_all.sh`
  without forcing the export into `data/conversations.json`

## Known Limits

- Alexko recovery depends on a private ChatGPT export or migrated historical corpus
- Gemini may still show misleading MCP status in `gemini mcp list --debug` even when
  direct tool calls succeed
- Codex may emit unrelated local sqlite rollout warnings during MCP runs
- this beta has been proven on the current fleet setup, not on a broad matrix of clean machines
- the current private repo history still contains internal development docs that should not ride along into a clean public first release
