# Project Godhand v0.1.0

**Date**: 2026-04-14
**Status**: First public release

## What This Release Is

The first public showcase of a 5-layer memory architecture for multi-LLM
systems. This release contains the complete Layer 2 (Lazarus) implementation
and architecture documentation for all 5 layers.

## What Works

### Layer 2: Lazarus (Included in This Repo)

- Local install bootstraps `.venv` + Qdrant via Docker
- 5 ingesters: ChatGPT, Claude Code, Gemini CLI, Codex CLI
- MCP server with 4 tools (summon, remember, rehydrate, stats)
- MCP registration for Claude, Gemini, and Codex
- Background daemon with launchd integration (optional)
- Export tool for LoRA/SFT training data
- Health check with drift validation (10.0/10)
- E2E integration tests (6/6 passing)

### Layer 1: MemPalace (Separate Package)

- v3.0.0, `pip install mempalace`
- 96.6% R@5 on LongMemEval
- 19 MCP tools
- ChromaDB backend

### Layers 3-5 (Documented)

- VexNet session sync protocol: documented with reference architecture
- Obsidian-Legion task coordination: documented with contract specification
- Furnace biometric integration: research preview with planned architecture

### Architecture Documentation (New in v0.1.0)

- 7 architecture documents covering all 5 layers + access control
- RENA proof: live demonstration of semantic resurrection (87ms across 28,714 exchanges)

## Upgrade from v0.1.0-beta.1

No breaking changes. This release adds architecture documentation, the 5-layer
narrative, and updated release metadata. All Layer 2 code is identical to
v0.1.0-beta.1.

## Known Limits

- Terminal-first onboarding (no GUI installer)
- Provider-side auth quirks may block otherwise healthy CLI integrations
- Gemini MCP status display can be misleading (tools still work)
- Tested on macOS only (should work on Linux, untested on Windows)

## Requirements

- Python 3.10+
- Docker (for Qdrant) or an existing Qdrant instance
- ~2GB disk for embedding model cache + vector data
